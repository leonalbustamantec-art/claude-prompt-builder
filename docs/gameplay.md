# Gameplay

Este documento define el ciclo principal del juego.

No describe la interfaz ni los componentes visuales. Su propósito es establecer cómo vive el usuario la experiencia de principio a fin.

Todas las funcionalidades del MVP deben reforzar este ciclo.

---

# 1. Objetivo del juego

El objetivo del jugador es construir una idea utilizando bloques visuales.

No existe una respuesta correcta.

El juego consiste en experimentar, modificar y descubrir cómo cambia el resultado al reorganizar las diferentes partes de una idea.

---

# 2. Ciclo principal

Todo el producto gira alrededor del siguiente ciclo:

**Explorar → Construir → Descubrir → Experimentar → Repetir**

Este ciclo nunca debe romperse.

Cada interacción debe motivar al usuario a iniciar una nueva construcción.

---

# 3. Explorar

El usuario comienza observando las opciones disponibles.

La aplicación debe despertar curiosidad desde el primer momento.

No debe existir una sensación de configuración inicial.

El usuario debe comprender que puede comenzar inmediatamente.

---

# 4. Construir

El usuario selecciona y combina bloques para construir una idea.

Cada bloque representa un concepto diferente.

Cada nueva combinación genera una construcción distinta.

El objetivo no es completar un formulario sino experimentar con posibilidades.

---

# 5. Descubrir

Una vez terminada la construcción, la aplicación genera un resultado.

Ese resultado representa la idea completa construida por el usuario.

El resultado debe incentivar inmediatamente la siguiente pregunta:

> ¿Qué ocurre si cambio una pieza?

---

# 6. Experimentar

El usuario modifica uno o varios bloques.

Observa el nuevo resultado.

Compara.

Aprende.

La experimentación constituye el verdadero proceso de aprendizaje.

---

# 7. Repetición

Cada construcción invita naturalmente a crear otra.

La aplicación nunca debe transmitir la sensación de haber terminado.

Siempre debe existir una razón para seguir explorando nuevas ideas.

---

# 8. Principios del gameplay

Todas las mecánicas deben favorecer:

* Curiosidad.
* Creatividad.
* Experimentación.
* Aprendizaje activo.
* Descubrimiento.

Nunca deben favorecer:

* Competencia.
* Frustración.
* Penalizaciones.
* Complejidad innecesaria.
* Memorización.

---

# 9. Restricciones del MVP

El gameplay del MVP no incluye:

* Niveles.
* Puntuaciones.
* Cronómetros.
* Recompensas.
* Inventarios.
* Progresión persistente.
* Misiones.

El único objetivo es validar que construir ideas mediante bloques resulte intuitivo, divertido y educativo.
