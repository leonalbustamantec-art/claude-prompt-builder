# Reglas de interacción

Este documento define cómo responde la aplicación a las acciones del usuario.

No describe el diseño visual. Define el comportamiento esperado durante la interacción.

Toda interacción debe reforzar la sensación de estar jugando con bloques físicos.

---

# 1. Principios generales

Toda interacción debe ser:

* Clara.
* Inmediata.
* Reversible.
* Intuitiva.
* Consistente.

El usuario nunca debe preguntarse qué acaba de ocurrir.

---

# 2. Los bloques

Cada bloque representa una única idea.

Su función debe comprenderse sin necesidad de leer documentación.

Los bloques deben incentivar la exploración.

Nunca deben intimidar al usuario.

---

# 3. Construcción

Toda construcción debe producir una respuesta visible.

Cada acción del usuario debe generar una sensación inmediata de progreso.

La aplicación debe comunicar constantemente que la idea está tomando forma.

---

# 4. Retroalimentación

Cada interacción debe responder de manera inmediata.

Ejemplos:

* Seleccionar un bloque.
* Mover un bloque.
* Agregar un bloque.
* Eliminar un bloque.
* Reemplazar un bloque.

Nunca deben existir acciones silenciosas.

---

# 5. Reversibilidad

El usuario debe poder modificar cualquier decisión.

Ninguna acción debe sentirse definitiva.

Experimentar debe ser sencillo y seguro.

---

# 6. Estados

La aplicación siempre debe comunicar claramente el estado actual de la construcción.

El usuario debe comprender:

* Qué ya construyó.
* Qué puede modificar.
* Qué resultado está obteniendo.

---

# 7. Errores

La aplicación debe minimizar la posibilidad de error.

Cuando ocurra una situación inesperada, la respuesta debe ser clara y amable.

Nunca utilizar mensajes técnicos.

Nunca bloquear la exploración.

---

# 8. Microinteracciones

Todas las microinteracciones deben tener un propósito.

Las animaciones deben comunicar:

* Selección.
* Movimiento.
* Cambio.
* Confirmación.
* Resultado.

Nunca deben existir animaciones únicamente decorativas.

---

# 9. Consistencia

Una misma acción siempre debe producir el mismo comportamiento.

Los usuarios deben poder construir confianza mediante la repetición.

La consistencia tiene prioridad sobre la innovación.

---

# 10. Criterio de calidad

Una interacción puede considerarse correcta cuando el usuario:

* Comprende inmediatamente qué ocurrió.
* Puede deshacer o modificar su decisión.
* Percibe una respuesta inmediata.
* Nunca necesita instrucciones para continuar.

Si una interacción requiere explicación adicional, probablemente deba simplificarse.
