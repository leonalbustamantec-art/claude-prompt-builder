# ui.md

## Dirección visual

La interfaz debe sentirse como un punto medio entre Scratch y un juguete de bloques físico.

Debe ser:

- Lúdica.
- Clara.
- Moderna.
- Infantil sin parecer para bebés.
- Fácil de implementar.

## Estructura

Una sola pantalla.

En escritorio:

- Selección de opciones a la izquierda.
- Construcción y resultado a la derecha.

En móvil:

- Todo en una sola columna.

## Bloques

Usar rectángulos con:

- Bordes redondeados.
- Sombra suave.
- Ícono o emoji.
- Texto breve.
- Un color por categoría.

No usar conectores complejos ni drag and drop.

## Categorías y colores

- Personaje: morado `#7C5CFC`
- Acción: naranja `#FF8A3D`
- Lugar: verde `#2FBF9B`
- Estilo: amarillo `#F4C542`

Colores base:

- Fondo: `#F7F8FC`
- Superficies: `#FFFFFF`
- Texto: `#1F2937`

## Tipografía

Usar `Nunito`, `Nunito Sans` o `system-ui`.

Debe ser grande, redondeada y fácil de leer.

## Interacción visual

Cuando una opción se selecciona:

- cambia al color principal de su categoría;
- se marca como activa;
- aparece en la zona de construcción;
- el resultado se actualiza inmediatamente.

Usar solo animaciones CSS breves y simples.

## Resultado

Mostrar una tarjeta con:

- El texto generado.
- Botón “Copiar idea”.
- Acción “Empezar de nuevo”.

## Restricciones

No implementar:

- Ilustraciones personalizadas.
- Drag and drop.
- Modo oscuro.
- Sonidos.
- Confeti complejo.
- Animaciones avanzadas.
- Más de una pantalla.
- Librerías visuales innecesarias.

## Regla final

Si una decisión visual aumenta mucho el tiempo o el costo de implementación, elegir la alternativa más simple.
